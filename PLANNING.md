# PRD: readme-gen — AI-Powered README Generator CLI

## Overview

`readme-gen` is an open-source CLI tool that scans a local codebase and automatically generates a high-quality, structured `README.md` file using AI. It requires zero configuration to get started and produces output good enough to ship on day one.

---

## Problem Statement

Most developers either skip writing a README, write a poor one, or spend disproportionate time on it. Existing tools are template-fillers that require manual input. There is no tool that reads the actual codebase and writes an accurate, useful README automatically.

---

## Goals

- Generate a complete, accurate README from any codebase in under 30 seconds
- Require zero config to run (`npx readme-gen` just works)
- Support all major languages and frameworks
- Produce output that is immediately usable, not a template to fill in
- Reach 1M+ monthly npm downloads and 5,000+ GitHub stars

---

## Non-Goals

- Not a documentation generator (no API docs, no JSDoc parsing)
- Not a wiki or multi-page doc tool
- Not a hosted SaaS — this is purely a local CLI

---

## Target Users

- Solo developers publishing open source projects
- Teams onboarding new contributors who need a README fast
- Hackathon participants who need polished docs quickly
- Anyone maintaining a repo with an outdated or missing README

---

## Core Features

### 1. Zero-Config Scan (`npx readme-gen`)

- Recursively scans the current directory
- Reads: `package.json`, `pyproject.toml`, `Cargo.toml`, `go.mod`, `composer.json`, or any other manifest
- Detects: language, framework, dependencies, scripts, entry points, license
- Excludes: `node_modules`, `.git`, `dist`, `build`, `.env` files automatically

### 2. AI-Powered Generation

- Sends a structured context payload (not raw code) to the AI model
- Context includes: file tree, manifest data, key source files (main entry, config files, README if exists)
- Generates a complete README with all standard sections
- Model: use Claude claude-sonnet-4-6 via the Anthropic API (user supplies `ANTHROPIC_API_KEY`)

### 3. README Sections Generated

The output README must include all applicable sections:

```
# Project Name
> One-line description

## Features
## Demo (if screenshots/GIFs detected in repo)
## Installation
## Usage (with code examples)
## Configuration (if config files detected)
## API Reference (if applicable)
## Contributing
## License
```

### 4. Smart File Selection

The tool should not send entire codebases to the AI. Instead:
- Always include: manifest files, entry point files, existing README
- Include up to 10 most relevant source files based on size and naming heuristics
- Truncate large files to first 200 lines
- Build a compact file tree (max depth 3)

### 5. Output Options

| Flag | Description |
|------|-------------|
| (default) | Writes `README.md` to current directory |
| `--print` | Prints to stdout instead of writing |
| `--update` | Merges with existing README instead of replacing |
| `--output <path>` | Write to a custom path |
| `--sections <list>` | Only generate specified sections |

### 6. Interactive Mode (optional `--interactive` flag)

If run with `--interactive`, prompt the user for:
- Project name (pre-filled from manifest)
- One-line description (pre-filled from manifest)
- Any extra context to pass to the AI (free text)

### 7. Diff Preview

- Before writing, show a diff of what will change if a README already exists
- Prompt user to confirm: `Write README.md? [Y/n]`
- Can be skipped with `--yes` flag

## Technical Specification

### Stack

- **Runtime**: Node.js 18+
- **Language**: TypeScript
- **CLI framework**: `commander` or `yargs`
- **AI SDK**: `@anthropic-ai/sdk`
- **File utilities**: Native `fs/promises`, `path`, `glob`
- **Output formatting**: `chalk` for terminal colors, `ora` for spinner

### Package Structure

```
readme-gen/
├── src/
│   ├── index.ts           # CLI entry point
│   ├── scanner.ts         # File system scanning and context building
│   ├── ai.ts              # Anthropic API call and prompt logic
│   ├── writer.ts          # README writing, diffing, confirmation
│   ├── detector.ts        # Language/framework detection
│   └── types.ts           # Shared TypeScript types
├── tests/
│   ├── scanner.test.ts
│   ├── detector.test.ts
│   └── fixtures/          # Sample project directories for testing
├── package.json
├── tsconfig.json
├── .eslintrc.json
└── README.md              # Generated by readme-gen itself
```

### AI Prompt Design

**System prompt:**
```
You are an expert technical writer. You generate README.md files for software projects.
Your output is always valid Markdown, professional, and accurate to the actual project.
Never invent features. Only describe what you can see in the provided context.
Do not include a preamble or explanation — output only the README content.
```

**User prompt structure:**
```
Generate a complete README.md for the following project.

Project Context:
- Name: {name}
- Language: {language}
- Framework: {framework}
- Dependencies: {dependencies}
- Scripts: {scripts}
- License: {license}

File Tree:
{fileTree}

Key Source Files:
{keyFiles}

{existingReadme ? "Existing README (update/improve this):" + existingReadme : ""}

Requirements:
- Include all applicable standard sections
- Use real code examples from the source files where possible
- Write installation and usage steps that actually work
- Keep the tone professional but approachable
```

### Language/Framework Detection Logic

| Detected file | Language | Framework |
|---|---|---|
| `package.json` with `"react"` dep | JavaScript/TypeScript | React |
| `package.json` with `"next"` dep | JavaScript/TypeScript | Next.js |
| `package.json` with `"express"` dep | JavaScript/TypeScript | Express |
| `pyproject.toml` or `requirements.txt` | Python | — |
| `requirements.txt` with `django` | Python | Django |
| `requirements.txt` with `fastapi` | Python | FastAPI |
| `Cargo.toml` | Rust | — |
| `go.mod` | Go | — |
| `composer.json` | PHP | — |
| `*.csproj` | C# | — |

## API Key Handling

- Read from `ANTHROPIC_API_KEY` environment variable
- Also accept `--api-key` flag (not recommended for security)
- If missing, print a helpful error with a link to get a key
- Never log or expose the key

---

## Error Handling

| Scenario | Behavior |
|---|---|
| No API key | Exit with clear message + link to Anthropic console |
| API call fails | Retry once, then exit with error message |
| Empty/unreadable directory | Exit with "No source files found" |
| Output file not writable | Exit with permission error message |
| User cancels at diff prompt | Exit cleanly with no changes |

## CLI Usage Examples

```bash
# Basic usage — scan current dir, write README.md
npx readme-gen

# Print to stdout only
npx readme-gen --print

# Update existing README instead of replacing
npx readme-gen --update

# Write to a different path
npx readme-gen --output docs/README.md

# Skip confirmation prompt
npx readme-gen --yes

# Interactive mode with extra context
npx readme-gen --interactive

# Only generate specific sections
npx readme-gen --sections installation,usage,contributing
```
